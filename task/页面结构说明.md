# 三农政策的乡村振兴职业画像测试

## 页面结构说明版（适合前端开发搭建）

## 1. 项目说明

本项目是一个职业画像测试型网页，主题为“三农政策的乡村振兴”。网页的核心流程是：

**首页 → 答题页 → 结果页 → 画像详情页 → 代表人物展示 → 情景模拟 → 情景详情页**

当前文档的目标不是填写完整内容，而是为前端开发提供清晰的页面结构、布局方式和模块划分，方便先搭建网页框架，后续再补充具体题目、画像、案例和情景内容。

---

## 2. 整体页面布局原则

整个网页建议采用**单页分区 + 跳转页面结合**的方式，保证结构清晰、适合展示，同时便于后续扩展。

### 2.1 布局风格
- 整体采用现代简洁风格
- 使用卡片式模块布局
- 页面层级清楚，重点内容突出
- 适合乡村振兴主题的视觉氛围
- 保留足够的内容扩展空间

### 2.2 页面宽度建议
- 桌面端内容居中显示
- 主内容区建议控制在 1200px 左右
- 移动端采用自适应布局
- 卡片和按钮在小屏幕下自动堆叠

### 2.3 统一组件建议
建议全站统一使用以下基础组件：
- 顶部导航栏 / 页面标题区
- 主视觉 Banner
- 卡片列表
- 按钮组
- 进度条
- 标签 / 关键词区域
- 折叠面板或弹窗
- 底部说明区

---

## 3. 页面结构总览

建议网页分为以下几个页面或模块：

1. 首页
2. 答题页
3. 测试结果页
4. 画像详情页
5. 代表人物模块

7. 情景详情页

每个页面的核心任务和布局如下所示。

## 4. 首页布局说明

### 4.1 页面目标
首页负责项目介绍和引导用户开始测试。

### 4.2 布局结构
首页建议分为 4 个区域：

#### A. 顶部标题区
- 显示项目名称
- 显示副标题或一句话介绍
- 可放一个简洁的主题图或插画

#### B. 主视觉区
- 放置项目的核心介绍
- 说明这是一个职业画像测试网站
- 放置“开始测试”按钮

#### C. 功能简介区
- 用 3 个左右的卡片介绍项目功能
- 例如：
  - 答题生成画像
  - 查看代表人物
  - 了解情景模拟

#### D. 底部说明区
- 放置项目主题说明
- 放置简短引导文案

### 4.3 首页布局示意

```text
┌──────────────────────────────┐
│ 项目标题 / 副标题             │
├──────────────────────────────┤
│ 主视觉区：介绍文案 + 按钮      │
├──────────────────────────────┤
│ 功能卡片区：3 个介绍模块       │
├──────────────────────────────┤
│ 底部说明区                    │
└──────────────────────────────┘
```

## 5. 答题页布局说明

### 5.1 页面目标
答题页用于展示测试题目并收集用户选择，是整个项目的核心交互部分。考虑到 PC 端页面宽度较大、单题分页容易造成视觉空缺，因此答题页应采用**一页多题、分区块展示**的方式。

### 5.2 布局结构
答题页建议分为 4 个区域：

#### A. 顶部信息区
- 显示当前测试名称
- 显示答题说明
- 显示答题进度
- 显示总题数或题目范围

#### B. 题目主体区
- 一页展示多道题目
- 每 2–3 题组成一个题组卡片
- 每题显示题目标题和多个单选项
- 题组之间保留间距，增强层次感

#### C. 操作按钮区
- 页面底部统一提供“提交测试”按钮
- 可保留“返回首页”按钮
- 如需分页浏览，可增加“上一组 / 下一组”切换，但不建议一题一页

#### D. 结果提示区
- 显示提交前提示
- 可提示用户当前还有多少题未完成
- 可在底部固定显示提交入口，方便操作

### 5.3 答题页布局示意

```text
┌──────────────────────────────────┐
│ 测试名称                         │
│ 简短说明 + 进度条                │
├──────────────────────────────────┤
│ 题组 1                           │
│ 题目 1                           │
│ ○ 选项 1  ○ 选项 2  ○ 选项 3     │
│ 题目 2                           │
│ ○ 选项 1  ○ 选项 2  ○ 选项 3     │
├──────────────────────────────────┤
│ 题组 2                           │
│ 题目 3                           │
│ ○ 选项 1  ○ 选项 2  ○ 选项 3     │
│ 题目 4                           │
│ ○ 选项 1  ○ 选项 2  ○ 选项 3     │
├──────────────────────────────────┤
│ 提交测试按钮 / 返回首页按钮      │
└──────────────────────────────────┘
```

### 5.4 答题页布局建议
- 采用一页多题布局，避免 PC 端页面过空
- 每道题使用卡片式或分块式结构
- 每个题组保持统一间距，保证阅读舒适
- 操作按钮放在页面底部，提升整体完整性
- 若题目数量较多，可使用滚动浏览，但仍保持同页展示

## 6. 测试结果页布局说明

### 6.1 页面目标
结果页用于展示用户测试生成的画像结果。

### 6.2 布局结构
结果页建议分为 5 个区域：

#### A. 结果标题区
- 显示“测试结果”
- 显示画像名称

#### B. 画像展示区
- 放置画像卡片或头像插图
- 作为页面视觉中心

#### C. 结果摘要区
- 一句话总结画像特征
- 显示关键词标签

#### D. 特征说明区
- 性格特点
- 能力特点
- 职业适配方向

#### E. 跳转操作区
- 查看详情按钮
- 返回测试按钮

### 6.3 结果页布局示意

```text
┌──────────────────────────────┐
│ 测试结果标题                 │
├──────────────────────────────┤
│ 画像卡片 / 头像               │
├──────────────────────────────┤
│ 一句话总结 + 关键词标签        │
├──────────────────────────────┤
│ 性格特点 / 能力特点 / 职业方向 │
├──────────────────────────────┤
│ 查看详情按钮                  │
└──────────────────────────────┘
```

### 6.4 结果页布局建议
- 重点突出画像名称
- 内容分块清晰
- 按钮放在显眼位置
- 结果区适合做成报告感页面

## 7. 画像详情页布局说明

### 7.1 页面目标
展示某一画像的完整介绍，让用户理解该画像的含义。

### 7.2 布局结构
画像详情页建议分为 6 个区域：

#### A. 顶部画像标题区
- 画像名称
- 简短定位语
- 背景图或插图

#### B. 画像简介区
- 画像的整体说明
- 核心特征介绍

#### C. 标签信息区
- 性格标签
- 能力标签
- 职业标签

#### D. 内容说明区
- 性格特征
- 职业适配
- 发展建议

#### E. 代表人物入口区
- 展示代表人物卡片
- 点击可查看人物详情

#### F. 情景模拟入口区
- 展示情景模块预览
- 点击可进入情景详情

### 7.3 画像详情页布局示意

```text
┌──────────────────────────────┐
│ 画像名称 + 定位语             │
├──────────────────────────────┤
│ 画像插图 / 背景图             │
├──────────────────────────────┤
│ 画像简介                      │
├──────────────────────────────┤
│ 标签区                        │
├──────────────────────────────┤
│ 性格 / 职业适配 / 建议        │
├──────────────────────────────┤

├──────────────────────────────┤

```

### 7.4 详情页布局建议
- 页面适合纵向滚动阅读
- 内容从上到下逐步展开
- 每个模块之间用分隔区分
- 重点内容采用卡片或大标题展示

## 8. 代表人物模块布局说明

### 8.1 页面目标
通过代表人物增强画像的真实感和可信度。

### 8.2 布局结构
代表人物模块建议采用卡片式横向排列或纵向排列。

每个代表人物卡片应包含：
- 人物头像 / 图片占位
- 人物姓名
- 简短简介
- 主要经历
- 对应画像原因

### 8.3 卡片布局示意

```text
┌──────────────┐  ┌──────────────┐
│ 人物图片     │  │ 人物图片     │
│ 姓名         │  │ 姓名         │
│ 简介         │  │ 简介         │
│ 对应原因     │  │ 对应原因     │
└──────────────┘  └──────────────┘
```

### 8.4 布局建议
- 桌面端可三列展示
- 移动端自动换行
- 卡片内容保持统一结构
- 如内容较多，可点击展开详情

## 9. 情景模拟模块布局说明

### 9.1 页面目标
展示画像在真实工作情境中的表现方式。

### 9.2 布局结构
情景模拟区建议采用卡片列表形式。

每个情景卡片包含：
- 场景名称
- 场景简介
- 简短提示
- 点击进入详情的入口

### 9.3 情景卡片布局示意

```text
┌──────────────────────────────┐
│ 场景名称                      │
│ 场景简介                      │
│ 点击查看详情                  │
└──────────────────────────────┘
```

### 9.4 布局建议
- 场景卡片按纵向列表排列
- 每个卡片内容简洁
- 进入详情后再展开具体分析
- 适合作为详情页后半部分内容
